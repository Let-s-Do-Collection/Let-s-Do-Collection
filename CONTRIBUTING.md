# Contributing

Thank you for your interest in contributing. Your help is greatly appreciated and helps keep this project growing and improving.

Pull Requests are welcome. Please take a moment to read the guidelines below before opening an issue or submitting changes.

---

## How You Can Help

### Pull Requests
- Make sure your changes are focused and well structured.
- Follow the existing code style and project structure.
- Ensure the project builds successfully before submitting.
- Keep changes clean and avoid unnecessary modifications.

### Feature Requests
- Open an issue describing your idea clearly.
- Explain the motivation and possible use cases.
- Be open to discussion and feedback.

### Bug Reports
- Provide a clear and detailed description.
- Include steps to reproduce the issue.
- Attach logs or screenshots when helpful.
- Specify the version and environment.

---

## General Guidelines

- Keep code clean, readable, and maintainable.
- Avoid leaving unused or commented-out code.
- Do not introduce unrelated refactors in the same pull request.
- Keep discussions respectful and constructive.

---

## Commit Messages

Clear and concise commit messages help everyone.

While not required, using the Conventional Commits format is encouraged:
https://www.conventionalcommits.org

Example:
fix(worldgen): resolve crash when loading world
feat(data): add new crafting recipe
chore(release): update changelog 

---

## Testing

- Test your changes thoroughly before submitting.
- Ensure no existing functionality is broken.
